# แผนพัฒนาแผนที่ Interactive เขตสุขภาพ -> จังหวัด -> อำเภอ

เป้าหมายคือพัฒนาแผนที่ใช้งานจริงในระบบ DHSD ให้ผู้ใช้สามารถคลิกเจาะข้อมูลได้ 3 ระดับ และเชื่อมกับข้อมูลประเด็นโรค/ภัยสุขภาพของพื้นที่นั้นโดยตรง

Flow หลัก:

```text
ประเทศไทย / 13 เขตสุขภาพ
  -> คลิกเขตสุขภาพ
จังหวัดในเขตสุขภาพที่เลือก
  -> คลิกจังหวัด
อำเภอในจังหวัดที่เลือก
  -> คลิกอำเภอ
แสดงประเด็นโรค/ภัยสุขภาพ + ข้อมูลล่าสุด + ทางเข้าฟอร์มตามสิทธิ์
```

## หลักการออกแบบ

- แผนที่ต้องเป็นส่วนหนึ่งของ Dashboard จริง ไม่ใช่ mockup แยก
- ใช้ master data เดิมของระบบให้มากที่สุด เช่น `master_agencies`, `master_provinces`, `master_districts`, `agency_provinces`, `intake_records`
- ใช้ GeoJSON สำหรับขอบเขตพื้นที่จริง
- โหลดข้อมูลเท่าที่จำเป็น เพื่อไม่ให้หน้าเว็บหนักเกินไป
- แยกการแสดงผล public/backoffice ตามสิทธิ์ของผู้ใช้
- ผู้ใช้ทั่วไปดูข้อมูลภาพรวมได้ แต่การกรอก/แก้ไขข้อมูลต้อง login และผ่าน role/scope

## Phase 1: ตรวจสอบข้อมูลและเตรียมโครงสร้างแผนที่

สถานะ: เริ่มดำเนินการแล้ว

เอกสาร/ไฟล์ที่สร้างใน Phase นี้:

- `phase-map/phase-1-map-readiness.md`
- `web/sql/017_phase_map_readiness_checks.sql`
- `web/types/map.ts`
- `web/services/map-boundary-service.ts`
- `web/public/map-boundaries/README.md`

วัตถุประสงค์:

เตรียมข้อมูลและโครงสร้างให้พร้อมก่อนเขียนแผนที่จริง

งานที่ต้องทำ:

- ตรวจสอบ `agency_provinces` ว่าจังหวัดทุกจังหวัดถูกผูกกับ สคร. ถูกต้อง
- ตรวจสอบ `master_provinces` และ `master_districts` ว่ารหัสจังหวัด/อำเภอตรงกับ GeoJSON
- เตรียมไฟล์ GeoJSON อย่างน้อย 2 ระดับก่อน: จังหวัด และอำเภอ
- กำหนด schema property ที่ต้องใช้ใน GeoJSON
- สร้าง service สำหรับ map boundary data

ข้อมูล GeoJSON ที่ต้องการ:

```text
provinces.geojson
- province_code
- province_name

districts.geojson
- district_code
- district_name
- province_code
```

หมายเหตุ:

ถ้าไม่มี GeoJSON เขตสุขภาพโดยตรง สามารถสร้างระดับเขตสุขภาพจากการ group จังหวัดตาม `agency_provinces` ก่อนได้

ผลลัพธ์ของ Phase:

- ข้อมูลจังหวัด/อำเภอพร้อมใช้งาน
- รหัสพื้นที่ในฐานข้อมูลกับ GeoJSON เชื่อมกันได้
- มีรายการจังหวัดที่ไม่มี สคร. หรืออำเภอที่ไม่มีจังหวัดให้แก้ก่อนขึ้น Phase ถัดไป

ผลตรวจปัจจุบัน:

- CSV master มี 77 จังหวัด และ 928 อำเภอ
- SQL mapping `005_full_agency_province_mapping.sql` มี 77 จังหวัดครบหลังเพิ่มอุดรธานีเข้า สคร.8
- มีไฟล์ GeoJSON จริงใน repo แล้วจาก `mapthai`
- `provinces.geojson` และ `districts/{province_code}.geojson` ถูก normalize แล้วเพื่อใช้ทำแผนที่ polygon จริงระดับมืออาชีพ

## Phase 2: สร้างแผนที่ระดับ 13 เขตสุขภาพ

สถานะ: เริ่มดำเนินการแล้ว

ไฟล์ที่เริ่มสร้างใน Phase นี้:

- `web/components/InteractiveHealthMap.tsx`
- `web/components/InteractiveHealthMapClient.tsx`

ข้อมูลแผนที่ที่นำมาใช้:

- `web/public/map-boundaries/provinces.geojson`
- `web/public/map-boundaries/districts/{province_code}.geojson`

วัตถุประสงค์:

สร้างหน้าแผนที่เริ่มต้นที่แสดง 13 เขตสุขภาพชัดเจนเหมือนภาพตัวอย่าง

งานที่ต้องทำ:

- สร้าง component ใหม่ เช่น `InteractiveHealthMap.tsx`
- แสดงแผนที่ประเทศไทยโดยแบ่งสีตาม 13 เขตสุขภาพ
- ใช้สีหลักให้เห็นเขตที่ hover/selected ชัดเจน
- เพิ่ม tooltip แสดงชื่อเขต เช่น `สคร.1`, `สคร.8`
- เพิ่ม legend สถานะแผนที่
- เพิ่ม breadcrumb เริ่มต้น `ประเทศไทย`
- คลิกเขตสุขภาพแล้วเปลี่ยนไป Phase view ระดับจังหวัด

พฤติกรรมผู้ใช้:

```text
เปิด Dashboard
เห็นแผนที่ 13 เขตสุขภาพ
hover เขต -> แสดงชื่อเขต
click เขต -> ซูมเข้าและแสดงจังหวัดในเขตนั้น
```

ผลลัพธ์ของ Phase:

- แผนที่ 13 เขตสุขภาพใช้งานได้จริง
- คลิกเขตแล้วรู้ว่าเลือกเขตอะไร
- Dashboard รับค่าพื้นที่ที่เลือกได้

## Phase 3: แสดงจังหวัดในเขตสุขภาพที่เลือก

สถานะ: เริ่มดำเนินการแล้ว

สิ่งที่ทำแล้ว:

- คลิกเขตสุขภาพแล้วแผนที่กรองเหลือเฉพาะจังหวัดในเขตนั้น
- เพิ่ม breadcrumb ระดับ `13 เขตสุขภาพ -> จังหวัดในเขต -> อำเภอ`
- เพิ่มปุ่ม `กลับสู่ 13 เขต`
- คลิกจังหวัดในเขตที่เลือกแล้ว highlight จังหวัด เพื่อเตรียมเข้าสู่ Phase 4
- เพิ่มข้อความสถานะจำนวนจังหวัดในเขตที่กำลังแสดง

วัตถุประสงค์:

เมื่อคลิกเขตสุขภาพ ให้แสดงเฉพาะจังหวัดในเขตนั้น พร้อมขอบเขตจังหวัดชัดเจน

งานที่ต้องทำ:

- กรองจังหวัดจาก `agency_provinces`
- แสดง polygon จังหวัดเฉพาะในเขตที่เลือก
- ซูม map ไปยัง bounds ของเขตนั้น
- เพิ่ม tooltip ชื่อจังหวัด
- เพิ่ม breadcrumb เช่น `ประเทศไทย > สคร.8`
- เพิ่มปุ่มย้อนกลับไป `13 เขตสุขภาพ`
- คลิกจังหวัดแล้วเปลี่ยนไป view ระดับอำเภอ

พฤติกรรมผู้ใช้:

```text
คลิก สคร.8
เห็นเฉพาะจังหวัดใน สคร.8
คลิก อุดรธานี
ระบบแสดงอำเภอในจังหวัดอุดรธานี
```

ผลลัพธ์ของ Phase:

- ผู้ใช้เจาะจากเขตสุขภาพไปจังหวัดได้
- จังหวัดแสดงตาม mapping จริงใน Supabase
- จังหวัดที่ไม่มี mapping ต้องไม่หลุดเป็น `-`

## Phase 4: แสดงอำเภอในจังหวัดที่เลือก

สถานะ: เริ่มดำเนินการแล้ว

สิ่งที่ทำแล้ว:

- คลิกจังหวัดแล้วโหลด GeoJSON อำเภอเฉพาะจังหวัดนั้นแบบ lazy loading
- แสดง polygon อำเภอแทน polygon จังหวัดเมื่อเข้าสู่ระดับจังหวัดที่เลือก
- เพิ่มปุ่ม `กลับสู่จังหวัดในเขต`
- tooltip อำเภอแสดงชื่ออำเภอและจังหวัด
- คลิกอำเภอแล้ว highlight อำเภอที่เลือก เพื่อเตรียม Phase 5
- เพิ่มข้อความสถานะจำนวนอำเภอในจังหวัดที่เลือก

วัตถุประสงค์:

เมื่อคลิกจังหวัด ให้แสดงอำเภอทั้งหมดในจังหวัดนั้น

งานที่ต้องทำ:

- โหลด GeoJSON อำเภอแบบ lazy loading ตาม `province_code`
- แสดง polygon อำเภอในจังหวัดนั้น
- เพิ่ม tooltip ชื่ออำเภอ
- เพิ่ม breadcrumb เช่น `ประเทศไทย > สคร.8 > อุดรธานี`
- เพิ่มปุ่มย้อนกลับไปจังหวัด/เขตสุขภาพ
- คลิกอำเภอแล้วเปิด detail panel

แนวทางโหลดข้อมูล:

```text
คลิกจังหวัด
  -> ส่ง province_code
  -> โหลด district GeoJSON เฉพาะจังหวัดนั้น
  -> render district polygons
```

เหตุผล:

GeoJSON อำเภอทั้งประเทศมีขนาดใหญ่ ถ้าโหลดทั้งหมดตั้งแต่แรกจะทำให้หน้าเว็บช้า

ผลลัพธ์ของ Phase:

- ผู้ใช้เจาะจากจังหวัดไปอำเภอได้
- แผนที่ยังโหลดเร็ว
- รองรับการต่อข้อมูลรายอำเภอใน Phase ถัดไป

## Phase 5: แสดงประเด็นโรค/ภัยสุขภาพรายอำเภอ

สถานะ: เริ่มดำเนินการแล้ว

สิ่งที่ทำแล้ว:

- คลิกอำเภอแล้วดึงข้อมูลจริงจาก `intake_records` ตาม `district_code`
- แสดงจำนวนรายการในอำเภอที่เลือก
- แสดงรายการ `ประเด็นโรค/ภัยสุขภาพ` ที่พบ
- แสดงข้อมูลล่าสุดสูงสุด 5 รายการ
- เพิ่ม empty state เมื่ออำเภอนั้นยังไม่มีข้อมูล
- แยก service `health-issue-service.ts` เพื่อเตรียมต่อสิทธิ์ใน Phase 6

วัตถุประสงค์:

เมื่อคลิกอำเภอ ให้แสดงข้อมูลจริงจากระบบตามพื้นที่นั้น

งานที่ต้องทำ:

- ดึงข้อมูลจาก `intake_records` ตาม `district_code`
- แสดง `ประเด็นโรค/ภัยสุขภาพ`
- แสดงรายการข้อมูลล่าสุดของอำเภอนั้น
- แสดงจำนวนรายการ/สถานะข้อมูล
- แสดงปุ่มเข้าสู่การกรอกข้อมูลถ้าผู้ใช้มีสิทธิ์
- ถ้ายังไม่ login ให้แสดงเฉพาะข้อมูล public

ตัวอย่างข้อมูลใน panel:

```text
อำเภอเมืองอุดรธานี
จังหวัดอุดรธานี
สคร.8

ประเด็นโรค/ภัยสุขภาพ:
- ไข้เลือดออก
- โรคติดต่อทางเดินหายใจ

ข้อมูลล่าสุด:
- วันที่บันทึก
- หน่วยงาน
- ผู้รายงาน
```

ผลลัพธ์ของ Phase:

- แผนที่เชื่อมกับข้อมูลจริง
- คลิกอำเภอแล้วเห็นประเด็นโรค/ภัยสุขภาพของพื้นที่นั้น
- พร้อมต่อเข้าฟอร์มหลังบ้าน

## Phase 6: เชื่อมสิทธิ์และฟอร์มหลังบ้าน

สถานะ: เริ่มดำเนินการแล้ว

สิ่งที่ทำแล้ว:

- ส่ง `AccessScope` เข้าแผนที่เพื่อให้รู้ role/scope ปัจจุบัน
- จำกัด polygon ที่แสดงตามจังหวัดที่ Dashboard อนุญาตให้เห็น
- แสดงข้อความ scope ใน detail panel รายอำเภอ
- แสดงปุ่ม `กรอกข้อมูลอำเภอนี้` เฉพาะผู้มีสิทธิ์ `canSubmitForOwnAgency`
- ปุ่มเลือกเขต/กลับ 13 เขตถูกซ่อนเมื่อผู้ใช้ถูกล็อกกับหน่วยงานของตัวเอง
- เมื่อกด `กรอกข้อมูลอำเภอนี้` ระบบเติม `agency/province/district` ลงฟอร์มหลังบ้านให้ทันที

วัตถุประสงค์:

ทำให้การดู/กรอกข้อมูลจากแผนที่เป็นไปตาม role ที่กำหนดไว้

สิทธิ์หลัก:

```text
superadmin
- เห็นทุกเขต ทุกจังหวัด ทุกอำเภอ
- เข้า dashboard เต็ม
- จัดการผู้ใช้ได้

admin
- เห็นข้อมูลภายใต้ สคร./scope ของตัวเอง
- กรอกข้อมูลของหน่วยงานตัวเอง

user
- เห็นและกรอกข้อมูลเฉพาะหน่วยงาน/พื้นที่ของตัวเอง
```

งานที่ต้องทำ:

- ใช้ `AccessScope` เดิมควบคุมพื้นที่ที่เห็น
- ซ่อน/ปิดการคลิกพื้นที่ที่อยู่นอก scope
- ถ้าคลิกอำเภอแล้วไม่มีสิทธิ์ ให้แสดงข้อมูล public เท่านั้น
- ปุ่มกรอกข้อมูลต้องเปิดเฉพาะผู้มีสิทธิ์
- กรอกข้อมูลแล้ว refresh แผนที่/summary อัตโนมัติ

ผลลัพธ์ของ Phase:

- แผนที่ทำงานร่วมกับ RBAC จริง
- ผู้ใช้แต่ละระดับเห็นพื้นที่ไม่เท่ากันตามสิทธิ์
- ลดความเสี่ยงข้อมูลข้ามหน่วยงาน

## Phase 7: ปรับ UI/UX ให้เป็นระดับมืออาชีพ

สถานะ: เริ่มดำเนินการแล้ว

สิ่งที่ทำแล้ว:

- breadcrumb เปลี่ยนเป็นปุ่มกดถอยระดับได้
- เพิ่ม loading overlay บนแผนที่ระหว่างโหลดจังหวัด/อำเภอ
- ปรับ detail panel รายอำเภอให้อ่านง่ายขึ้น
- ปรับ permission box ให้เห็นชัดว่าใครกรอกข้อมูลได้
- ปรับ responsive สำหรับจอเล็กให้ toolbar, permission box และ detail panel ไม่บีบกัน
- เพิ่ม animation loading spinner สำหรับ map state

วัตถุประสงค์:

ทำให้แผนที่ใช้งานง่ายและดูเป็นระบบงานจริง

งานที่ต้องทำ:

- เพิ่ม breadcrumb ที่ชัดเจน
- เพิ่มปุ่มย้อนกลับแต่ละระดับ
- เพิ่มปุ่ม `กลับสู่ประเทศไทย`
- เพิ่ม loading state ระหว่างโหลด GeoJSON
- เพิ่ม empty state เมื่อพื้นที่นั้นยังไม่มีข้อมูล
- เพิ่ม selected/hover style ที่ชัดเจน
- เพิ่ม legend และคำอธิบายสี
- เพิ่ม responsive layout สำหรับจอเล็ก
- ปรับ panel ให้แสดงข้อมูลสั้น กระชับ และใช้งานง่าย

โครง UI แนะนำ:

```text
┌──────────────────────────────────────────────┐
│ Breadcrumb: ประเทศไทย > สคร.8 > อุดรธานี     │
├───────────────────────────┬──────────────────┤
│                           │ Detail Panel      │
│        Interactive Map     │ ข้อมูลพื้นที่     │
│                           │ ประเด็นโรค        │
│                           │ ปุ่มกรอกข้อมูล    │
└───────────────────────────┴──────────────────┘
```

ผลลัพธ์ของ Phase:

- ผู้ใช้เข้าใจว่าตัวเองอยู่ระดับไหน
- ใช้งานแผนที่ได้โดยไม่หลง
- พร้อมนำเสนอเป็นระบบจริง

## สิ่งที่ต้องตัดสินใจก่อนเริ่มเขียนโค้ด

- มี GeoJSON จังหวัดและอำเภอพร้อมหรือยัง
- GeoJSON ใช้รหัสจังหวัด/อำเภอตรงกับ master data หรือไม่
- ต้องการให้แผนที่เดิมถูกแทนที่ทันที หรือทำ component ใหม่แล้วค่อยสลับ
- ระดับ public จะเห็นข้อมูลโรค/ภัยสุขภาพละเอียดแค่ไหน
- admin/user จะถูกจำกัดด้วย agency, province หรือ district เป็นหลัก

## ข้อเสนอสำหรับระบบนี้

ให้สร้างเป็น feature จริงแยก component ใหม่ก่อน แล้วนำไปแทนแผนที่เดิมใน Dashboard แบบค่อยเป็นค่อยไป วิธีนี้ปลอดภัยที่สุด เพราะ component เดิมยังเก็บไว้เป็น fallback/reference และเราค่อยยกระดับแผนที่ใหม่ด้วย Phase 3-4 ต่อไป
