

ระบบวิเคราะห์และประมวลผลข้อมูลการส่งงานรายอำเภอ

(สำหรับใช้เป็นคู่มือคำสั่ง/Prompt เพื่อให้ AI ทำความเข้าใจและเขียนโปรแกรมระบบแสดงผล)

1. วัตถุประสงค์ (Objective)

ให้ AI ทำความเข้าใจกฎเกณฑ์การคำนวณ และโครงสร้างข้อมูลสำหรับการแสดงผลกราฟแท่งแบบสะสมแนวนอน (Horizontal Stacked Bar Chart) เพื่อนำเสนออัตราส่วนความคืบหน้าการส่งงานของอำเภอทั้งหมดในพื้นที่อย่างถูกต้องและเป็นสัดส่วนแบบ Real-time

2. หลักการและสูตรคำนวณทางคณิตศาสตร์ (Mathematical Logic)

กำหนดให้:

$N$ คือ จำนวนอำเภอทั้งหมดในเป้าหมาย (ในโจทย์นี้ตั้งค่าเริ่มต้นไว้ที่ $5$ อำเภอ)

$S$ คือ จำนวนอำเภอที่ส่งงานเรียบร้อยแล้ว (หน่วย: อำเภอ)

$P$ คือ จำนวนอำเภอที่ยังค้างส่งงาน (หน่วย: อำเภอ)

กฎการจัดสรรน้ำหนัก (Weight Allocation):

งานทั้งหมดคิดเป็น $100\%$ ดังนั้น น้ำหนักความสำคัญของแต่ละอำเภอจะถูกแบ่งเท่าๆ กันตามสูตร:


$$\text{น้ำหนักต่อ 1 อำเภอ} = \frac{100\%}{N}$$

กรณี $N = 5$ จะได้:


$$\text{น้ำหนักต่อ 1 อำเภอ} = \frac{100\%}{5} = 20\%$$

สูตรคำนวณเปอร์เซ็นต์ความคืบหน้า:

เปอร์เซ็นต์การส่งงานสำเร็จ (Sent Percentage - $Pct_{sent}$):


$$Pct_{sent} = \left( \frac{S}{N} \right) \times 100$$


ตัวอย่างหากส่งแล้ว 3 อำเภอ:


$$Pct_{sent} = \left( \frac{3}{5} \right) \times 100 = 60\%$$

เปอร์เซ็นต์การค้างส่งงาน (Pending Percentage - $Pct_{pending}$):


$$Pct_{pending} = \left( \frac{P}{N} \right) \times 100 \quad \text{หรือ} \quad Pct_{pending} = 100\% - Pct_{sent}$$


ตัวอย่างหากค้างส่ง 2 อำเภอ:


$$Pct_{pending} = \left( \frac{2}{5} \right) \times 100 = 40\%$$

3. รูปแบบโครงสร้างข้อมูลนำเข้า (Input Data Schema)

เมื่อระบบหลังบ้าน (Backend) หรือผู้ใช้ส่งข้อมูลเข้ามา ให้ใช้รูปแบบ JSON Array of Objects ดังนี้ในการประมวลผล:

[
  { "id": 1, "name": "อ.เมือง", "sent": true },
  { "id": 2, "name": "อ.สันทราย", "sent": true },
  { "id": 3, "name": "อ.แม่ริม", "sent": true },
  { "id": 4, "name": "อ.หางดง", "sent": false },
  { "id": 5, "name": "อ.สันกำแพง", "sent": false }
]


4. รูปแบบโครงสร้างข้อมูลผลลัพธ์ (Expected Output Schema)

AI จะต้องประมวลผลข้อมูลนำเข้าข้างต้น แล้วส่งออกเป็นผลลัพธ์ (State) สำหรับนำไปวาดกราฟ ดังนี้:

{
  "summary": {
    "totalDistricts": 5,
    "sentCount": 3,
    "pendingCount": 2,
    "sentPercent": 60,
    "pendingPercent": 40
  }
}


5. กฎการแสดงผลบนหน้าต่างผู้ใช้งาน (UI Rendering Rules)

เพื่อให้กราฟสะสมแนวนอนแสดงผลได้อย่างเป็นมืออาชีพและไม่มีข้อผิดพลาดด้านการแสดงผล (Visual Bugs) AI ต้องปฏิบัติตามเงื่อนไขต่อไปนี้:

โครงสร้างกราฟ (Stacked Bar Layout):

ต้องใช้ระบบ Flexbox ในการเรียงตัว โดยมีคอนเทนเนอร์หลักความกว้าง $100\%$ (width: 100%)

แถบแสดงผลก้อนที่ 1 (ส่งแล้ว) ให้กำหนดความกว้างตามค่าความกว้างเปอร์เซ็นต์ที่คำนวณได้ ($Pct_{sent}$)

แถบแสดงผลก้อนที่ 2 (ค้างส่ง) ให้กำหนดความกว้างตามค่าความกว้างเปอร์เซ็นต์ที่คำนวณได้ ($Pct_{pending}$)

การกำหนดธีมสี (Theme Colors):

ฝั่งส่งงานสำเร็จ (Sent): ใช้สีเขียวโทนสว่าง เช่นสี Emerald Green (#10b981 หรือ #2ecc71) เพื่อสื่อถึงความสำเร็จ

ฝั่งค้างส่ง (Pending): ใช้สีแดงโทนสว่าง เช่นสี Rose Red (#ef4444 หรือ #e74c3c) เพื่อเป็นจุดสังเกตสำหรับติดตามงาน

ระบบป้องกันข้อความทับซ้อน (Label Collision Guard):

หากสัดส่วนฝั่งใดฝั่งหนึ่งมีค่าน้อยเกินไป (เช่น $\le 10\%$) ตัวเลขเปอร์เซ็นต์อาจล้นขอบหรือเบียดทับกันได้

กฎ: หากค่าเปอร์เซ็นต์ของส่วนนั้น ๆ น้อยกว่าหรือเท่ากับ $10\%$ ให้ซ่อนตัวอักษรเปอร์เซ็นต์ข้างในแท่งกราฟนั้นเสีย (หรือนำไปแสดงผลด้านนอกแทน)

การตอบสนองแบบยืดหยุ่น (Responsive Design):

แท่งกราฟและอัตราส่วนต้องยืดหดตามขนาดหน้าจอ (Responsive) โดยหลีกเลี่ยงการกำหนดขนาดความกว้างแบบฟิกพิกเซล (Fixed width) ให้เลือกใช้เปอร์เซ็นต์หรือหน่วยสัมพันธ์เป็นหลัก

6. ตัวอย่างโค้ดโครงสร้างเริ่มต้น (Boilerplate Sample)

หากต้องการให้สร้างหน้าเว็บแสดงผลในรูปแบบ CSS Flexbox ให้ใช้โครงสร้างดังนี้:

<div style="display: flex; width: 100%; height: 30px; border-radius: 8px; overflow: hidden;">
    <!-- แท่งเขียว -->
    <div style="width: 60%; background-color: #10b981; display: flex; align-items: center; justify-content: center; color: white;">60%</div>
    <!-- แท่งแดง -->
    <div style="width: 40%; background-color: #ef4444; display: flex; align-items: center; justify-content: center; color: white;">40%</div>
</div>
